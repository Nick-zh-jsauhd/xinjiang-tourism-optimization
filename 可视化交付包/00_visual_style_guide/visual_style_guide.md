# 可视化风格规范

## 字体

论文和答辩图建议使用：

| 元素 | 字体 | 字重 |
|---|---|---|
| 中文标题 | 宋体 SimSun | 加粗 |
| 中文坐标轴标签 | 宋体 SimSun | 加粗 |
| 密集中文节点/表格文字 | 宋体 SimSun | 常规或半粗 |
| 英文/数字 | Times New Roman | 常规 |
| 英文标题/模型缩写 | Times New Roman | 加粗 |

注意：中文全部加粗会让地图节点和热力表过密。标题、轴标签和关键注释加粗即可，密集标签以可读性优先。

## 配色

使用 `color_palette.csv` 中的固定色值。风险表达遵循：

| 状态 | 颜色 |
|---|---|
| 低风险/通过 | 绿色 |
| 中等风险/需微调 | 橙色 |
| 高风险/失败 | 红色 |
| 缓冲/背景 | 灰色 |

四问主题色：

| 问题 | 主题色 | 用途 |
|---|---|---|
| Q1 | 深蓝 | 个人旅游路线与鲁棒决策 |
| Q2 | 绿色 | 两年覆盖与费用节省 |
| Q3 | 紫色 | 文化考察与专项调度 |
| Q4 | 青蓝 | 节假日容量运营 |

## 标题

标题优先使用结论句：

- 推荐：`32景点是覆盖上界，24景点才是运营鲁棒主推`
- 不推荐：`Q1路线对比图`

## 图表纪律

- 不用颜色制造无意义分类。
- 不用折线图连接离散策略。
- 同一张图不混合候选池和代表方案。
- 成功率图必须说明口径。
- 地图图层不能压过路线和节点。
- 热力图必须显示单位、阈值和资源约束口径。

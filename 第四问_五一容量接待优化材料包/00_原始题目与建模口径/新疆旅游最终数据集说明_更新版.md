# 新疆旅游线路安排最终数据集说明（更新版）

## 1. 数据集位置

本项目当前没有把所有数据压成一个单表，而是按“原始题面数据 -> 基础清洗数据 -> 增强建模数据 -> 实验结果证据”的层级组织。

完整数据集应以以下目录和文件共同构成：

| 层级 | 位置 | 作用 |
|---|---|---|
| 原始题面数据 | `新疆旅游全品类数据明细.xlsx` | 题目提供的景点、费用、交通、住宿等原始数据来源 |
| 基础清洗数据 | `model_data/` | 将原始 Excel 拆解为可建模的景点、枢纽、道路、酒店、费用和质量问题表 |
| 增强建模数据 | `enhanced_data/` | 当前四问模型实际使用的主数据层，包含高德 OD、多模式交通、时间窗、容量、住宿、文化标签和情景参数 |
| 最终结果与证据 | `outputs/` 与各 `*_outputs/` | 各问求解结果、仿真实验、风险策略、图表和答辩材料 |
| 最终证据注册表 | `final_research_package_v2_outputs/` | 数据真实性、证据链、论文插入点和最终答辩故事线 |

若只交付一个“最终建模数据包”，建议打包以下三类目录：

1. `model_data/`
2. `enhanced_data/`
3. `final_research_package_v2_outputs/`

并附带主要结果文件：

- `outputs/新疆旅游最终科研交付总包v2.md`
- `outputs/新疆旅游最终科研交付总包v2.xlsx`
- `outputs/新疆旅游建模答辩终版v2.pptx`

## 2. 基础清洗数据层：`model_data/`

`model_data/` 是从题面 Excel 和早期文本数据中整理出的基础表，主要用于构造最初的景点-枢纽-道路图。

| 文件 | 行数 | 内容 |
|---|---:|---|
| `spot_clean.csv` | 40 | 40 个景点的名称、区域、枢纽、游览时长、门票、风险和建模属性 |
| `hub_clean.csv` | 25 | 交通/住宿/区域枢纽节点 |
| `spot_hub_map.csv` | 40 | 景点到最近建模枢纽的映射 |
| `edge_road_directed.csv` | 56 | 早期道路有向边，主要作为基础图和对照 |
| `transport_cost_clean.csv` | 33 | 原始交通费用清洗表 |
| `hotel_options_clean.csv` | 29 | 酒店选项与高低季价格 |
| `hotel_place_default.csv` | 19 | 地点级默认住宿价格 |
| `scenario_parameters.csv` | 10 | 基础场景参数 |
| `data_quality_issues.csv` | 20 | 数据质量问题和修复建议 |
| `source_sites.csv` | 12 | 原始来源说明 |
| `data_dictionary.csv` | 9 | 基础表含义和优化用途 |

这一层的作用是“把题面数据变成结构化建模表”。它不是最终最强交通数据层，因为早期道路边仍较粗糙。

## 3. 增强建模数据层：`enhanced_data/`

`enhanced_data/` 是当前最重要的数据目录。后续四问建模、逐日排程、仿真和风险策略都应优先引用这一层。

### 3.1 景点与空间数据

| 文件 | 行数 | 内容 |
|---|---:|---|
| `spot_coordinates_amap.csv` | 40 | 40 个景点的高德坐标、地址、POI 匹配质量 |
| `spot_time_windows.csv` | 40 | 景区开放时间窗与规则质量 |
| `special_access_constraints.csv` | 40 | 审批、边防证、向导、越野车、普通游客可达性 |
| `cultural_tags.csv` | 40 | 丝路、宗教、民俗、考古、自然等标签和文化价值分 |

### 3.2 交通网络数据

| 文件 | 行数 | 内容 |
|---|---:|---|
| `amap_driving_od_matrix_clean.csv` | 1600 | 40x40 景点间高德驾车 OD，含距离、时长、费用、风险 |
| `amap_depot_access_matrix_clean.csv` | 80 | 乌鲁木齐到各景点、各景点回乌鲁木齐的高德驾车接驳 |
| `multimodal_nodes.csv` | 108 | 景点、枢纽、机场、火车站、交通节点 |
| `multimodal_edges.csv` | 1900 | 自驾、铁路、航班、接驳、景区交通等多模式边 |
| `enhanced_od_matrix.csv` | 1600 | 景点到景点的多模式最短路闭包，含时间、费用、风险、路径模式 |
| `enhanced_od_matrix_with_amap.csv` | 1600 | 多模式闭包叠加高德驾车 OD 字段，便于道路层对照 |
| `depot_access_matrix.csv` | 40 | 乌鲁木齐起终点接驳闭包，当前主要用于第一问和部分闭合路线 |
| `edge_time_scenarios.csv` | 9500 | 不同天气、节假日、道路扰动下的边时间、费用和可用性 |
| `time_dependent_rules.csv` | 15 | 场景概率、时间因子、费用因子、风险增量 |

需要特别说明：`enhanced_od_matrix.csv` 已不是纯高德自驾矩阵，而是多模式交通闭包。整表 1600 条 OD 中，路径模式包含 `self_drive`、`scenic_shuttle`、`rail`、`air`、`transfer` 等。但具体模型求解出的路线是否使用铁路/航班，要看路线段输出表。

### 3.3 铁路与航班数据

| 文件 | 行数 | 内容 |
|---|---:|---|
| `rail_12306_raw_core_od.csv` | 132 | 12306 核心铁路 OD 原始查询结果 |
| `rail_12306_model_options.csv` | 24 | 从铁路原始结果中筛选出的最快、最便宜、夜间/代表班次 |
| `air_public_schedule_seed.csv` | 27 | 公开航班时刻与价格代理快照 |

铁路数据可信度相对更高，已来自 12306 查询口径；航班数据属于公开时刻/报价快照或代理变量，不应写成实时机票价格。

### 3.4 住宿、容量与仿真参数

| 文件 | 行数 | 内容 |
|---|---:|---|
| `hotel_hub_constraints.csv` | 25 | 枢纽住宿可得性、默认房价、模拟房量 |
| `capacity_by_spot.csv` | 40 | 景区日容量、瞬时容量和有效容量系数 |
| `data_gap_acquisition_plan.csv` | 9 | 仍需补齐的数据、来源和模型用途 |
| `source_registry.csv` | 20 | 数据来源注册表 |
| `model_status_recommendations.csv` | 7 | 各模块当前状态、限制和下一步 |

酒店房量、景区容量和预约余量目前主要是校准/仿真参数，不是实时平台数据。

## 5. 数据真实性分层

| 数据类别 | 当前状态 | 论文/答辩表述 |
|---|---|---|
| 景点名称、门票、游览时长、偏好属性 | 题面 Excel 清洗 | 可作为本案例基础主数据 |
| 高德驾车 OD 距离/时长 | 高德 API 快照 | 可作为真实道路层边权，但不是逐小时实时拥堵 |
| 多模式交通闭包 | 高德道路 + 铁路候选 + 航班种子 + 接驳规则生成 | 可作为增强建模 OD，需说明铁路/航班不是完整实时班次网络 |
| 12306 铁路 | 核心 OD 查询与代表班次筛选 | 可作为铁路层样例数据，正式出行仍需复核余票 |
| 航班 | 公开时刻/报价快照或代理变量 | 只能作为航班层种子，不能声称实时票价 |
| 酒店价格与房量 | 价格清洗 + 房量模拟 | 适合作为住宿约束和风险仿真参数 |
| 景区容量/预约余量 | 规则校准 + 仿真 | 可做容量流和压力测试，不能称为实时预约余量 |
| 天气/封路/审批延误 | 情景模拟 | 用于数字孪生与鲁棒性，不是预测模型 |

## 6. 推荐引用口径

论文或答辩中建议这样描述：

> 本研究的数据集由题面 Excel 清洗数据、真实高德驾车 OD 快照、多模式交通网络、12306 核心铁路 OD、公开航班时刻种子、住宿与容量校准参数共同组成。基础景点属性来自题目附件并结构化清洗；道路层使用高德 API 的景点级 OD；铁路和航班作为多模式交通增强层；酒店房量、景区容量、预约余量、天气和道路封闭风险属于场景校准或仿真参数。最终模型并非实时商业调度系统，而是一个可复现、可替换数据源的图论与运筹优化研究数据底座。

## 7. 当前最终结论

当前项目中，最完整、最该作为“最终数据集”的不是某一个 Excel，而是：

```text
model_data/
enhanced_data/
final_research_package_v2_outputs/
各问题结果目录：
  hybrid_30day_outputs/
  problem2_openpath_outputs/
  enhanced_model_outputs/
  digital_twin_outputs/
  risk_policy_outputs/
```

其中，`enhanced_data/` 是模型输入主数据；`model_data/` 是基础清洗来源；`final_research_package_v2_outputs/` 是数据真实性和证据链说明；各 `*_outputs/` 是实验结果。

# 新疆旅游线路优化：第一问 30 天路线优化材料包

本仓库只保留“新疆旅游线路安排”项目中第一问相关的材料、数据、代码快照和实验结果。

第一问目标：

> 在 30 天暑期周期内，为王先生夫妇设计一组兼顾景点覆盖、费用控制、交通可达、住宿落地、体力舒适和风险缓冲的新疆旅游路线方案。

本仓库当前重点是第一问 V2.1 强化版：将原来的单条 32 景点硬 30 天路线，升级为“epsilon-coverage 候选路线族 + 后验稳健性评价”。当前版本不是严格 Pareto 前沿重优化，也不是完整 chance-constrained 主模型。

## 当前结论

| 方案 | 定位 | 景点 | 天数 | 缓冲 | 建议 |
|---|---:|---:|---:|---:|---|
| 极限覆盖版 | 对照基准 | 32 | 30 | 0 | 展示覆盖上界，不作为现实主推 |
| 均衡稳健版 | 主推 | 30 | 28 | 2 | 普通游客现实推荐方案 |
| 亲子舒适版 | 扩展 | 30 | 30 | 0 | 降低每日强度 |
| 长者慢游版 | 扩展 | 20 | 30 | 7 | 控制高海拔、长转场和连续疲劳 |

论文表述建议：

> 32 景点路线是硬约束下的覆盖上界；现实推荐方案为 30 景点、28 天有效行程、2 天机动缓冲的均衡稳健版。

答辩时需要同时说明：楼兰古城因审批和普通游客限制不进入基准自由行主线，模型采用可普通访问的西域文化/丝路文化节点进行替代满足。

## 环境配置

建议使用 Python 3.10+。

```bash

```

如需重新调用高德 API，复制环境变量模板：

```bash
copy .env.example .env
```

真实 `.env` 文件不应提交到 GitHub。

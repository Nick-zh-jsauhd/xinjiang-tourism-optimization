# 新疆旅游线路安排四问最终收束方案

## 0. 总体收束结论

本案例已经可以收束为一个完整的图论与运筹优化建模方案。最终不建议继续把所有算法并列展示，而应采用“主问题四问 + 数据强化 + 算法拓展”的层次：

| 模块 | 正文主结果 | 用途 |
|---|---|---|
| 第一问：30天低成本多景点游 | 30天硬约束混合元启发式 `HYBRID_30D_ACO_ALNS_SA` | 原题主答案 |
| 第一问基准 | `PCOP_MILP` 修复方案 | 严格数学模型基准 |
| 第二问：两年暑假游完新疆 | 两路径开放覆盖 + 新疆境内交通费用最小化 | 原题主答案 |
| 第三问：三组文化考察 | 最小最大完成时间多旅行商 `S3_MinMax` | 文化专项均衡方案 |
| 第四问：五一12天接待线路 | 候选线路生成 + 容量流分配 | 群体接待与容量配置方案 |
| 拓展 | 联合优化、随机仿真、开放时间混合元启发式 | 答辩亮点和鲁棒性说明 |

最终汇报的核心逻辑应是：先用真实与校准数据构造“景点-交通-时间-容量”的加权图，再针对四类决策场景分别建立不同的图优化模型。第一问解决个人旅游路线选择，第二问解决跨年份覆盖，第三问解决多团队文化考察均衡，第四问解决节假日客流分配。

## 1. 数据与图结构收束

本案例最终使用的基础图为有向加权图：

- 节点：景点、交通枢纽、住宿枢纽。
- 边：景点间交通连接、起终点接驳、同城/同区域接驳。
- 边权：时间、费用、风险、换乘便利度。
- 节点权：景点价值、门票、游览时长、文化价值、预约容量、开放时间。

数据真实性应在报告中分层说明：

| 数据类别 | 当前状态 | 汇报口径 |
|---|---|---|
| 景点名称、门票、游览时长、主题属性 | 来自原始Excel清洗 | 可作为本案例基础数据 |
| 高德驾车OD | 已调用高德API获得真实道路时间与距离 | 可作为真实交通OD |
| 起终点接驳OD | 已接入高德API | 可作为真实接驳时间/成本 |
| 12306铁路 | 已采集核心OD种子，原始132条，模型选项24条 | 作为铁路公开时刻种子，不等同实时余票 |
| 航班 | 已构建公开航班时刻种子27条 | 作为航班可达性种子，不等同实时票价 |
| 酒店房量/价格 | 模型校准/仿真 | 不能称为真实实时房态 |
| 景区容量/预约余量 | 模型校准/仿真 | 用于场景模拟和敏感性分析 |
| 客流扰动、天气、预约失败 | Monte Carlo仿真 | 用于鲁棒性检验 |

因此，本案例不是实时商业系统，而是“真实交通OD + 校准容量场景 + 运筹优化算法”的课程建模案例。这个定位是合理且可答辩的。

## 2. 第一问：30天内低成本多景点旅游路线

### 2.1 问题收束

第一问本质是带收益的路径规划问题。游客希望在有限时间内尽可能多游览高价值景点，同时控制交通、门票、住宿成本，并满足开放时间、每日活动时长、住宿可达性等约束。

可将其抽象为 Prize-Collecting Orienteering Problem，简称 PCOP：

- 每个景点有收益 `v_i`，包括景点评分、主题偏好、文化价值等。
- 每条边有时间 `t_ij`、成本 `c_ij`、风险 `r_ij`。
- 路线从起点出发，访问若干景点后返回或到达终点。
- 总天数必须不超过30天。

### 2.2 主模型

定义：

- `x_i = 1` 表示选择景点 `i`。
- `y_ij = 1` 表示路线中从 `i` 到 `j`。
- `D` 表示逐日排程后的总天数。
- `W` 表示时间窗违规次数。
- `L` 表示住宿容量或住宿枢纽不足次数。

目标函数采用收益最大化与成本惩罚结合：

```text
max  sum(v_i x_i)
     - alpha * transport_cost
     - beta  * ticket_cost
     - gamma * hotel_cost
     - eta   * over_budget_days
     - theta * time_window_violations
     - mu    * limited_lodging_nights
```

核心约束：

```text
sum_j y_ij = x_i
sum_j y_ji = x_i
D <= 30
W = 0
每日活动时间 <= 标准日预算，超出部分记入惩罚
路线不得重复访问同一景点
普通游客不可达景点不纳入第一问候选集
```

第一问最终采用两层求解：

1. `PCOP_MILP` 给出严格数学模型基准。
2. `HYBRID_30D_ACO_ALNS_SA` 在全候选集上做30天硬约束搜索，作为正文主推方案。

混合元启发式由三部分组成：

- ACO 蚁群算法负责构造高质量初始路线。
- ALNS 自适应大邻域搜索负责破坏与修复路线。
- SA 模拟退火接受准则负责跳出局部最优。

### 2.3 最终主结果

第一问正文主推路线为 `HYBRID_30D_ACO_ALNS_SA`：

| 景点数 | 32 |
| 排程天数 | 30 |

| 交通费用，两人 | 5516.53元 |

| 不含餐饮代理总成本 | 14548.53元 |

路线为：

### 2.4 基准与拓展对比

### 2.5 第一问答辩口径

第一问最终不再使用开放时间版作为主结果。主结果是30天硬约束混合元启发式；`PCOP_MILP` 是可解释基准；开放时间版用于说明算法在放宽期限时可进一步提升覆盖率。

## 3. 第二问：两年暑假游完新疆

### 3.1 问题收束

第二问题面为：“如果他们打算今、明两年暑假完成对新疆的旅游，请为他们设计合适的旅游路线，使在新疆境内的交通费用尽量节省。”因此，本问的主目标不是景点数均衡、年份天数均衡或文化价值最大化，而是：

```text
在两个暑假分别形成一条新疆境内旅游路径，覆盖普通游客可达景点，并最小化新疆境内交通费用。
```

这里普通游客候选集为38个景点。楼兰古城、尼雅遗址属于特殊文化考察点，因需审批、成本和可达性特殊，放入第三问文化专项，不作为第二问普通游客主线路；若题目要求“所有给定点均覆盖”，可作为审批扩展方案另列。

### 3.2 主模型

第二问最终应建模为“两条开放路径覆盖问题”，而不是简单谱聚类。谱聚类可以作为初始分区启发式，但最终模型必须围绕“新疆境内交通费用最小”展开。

设：

- `V` 为普通游客可达景点集合。
- `K = {1, 2}` 表示今、明两个暑假。
- `G` 为新疆境内可选入疆/离疆枢纽集合，例如乌鲁木齐、伊宁、阿勒泰、喀什、库尔勒等。
- `c_ij` 为景点 `i` 到景点 `j` 的新疆境内交通费用。
- `c_gi` 为从新疆境内枢纽 `g` 到景点 `i` 的接驳费用。
- `c_ig` 为景点 `i` 到新疆境内枢纽 `g` 的离疆接驳费用。

- `x_ik = 1` 表示景点 `i` 分配给第 `k` 年。
- `y_ijk = 1` 表示第 `k` 年路线从 `i` 到 `j`。
- `s_gk = 1` 表示第 `k` 年从枢纽 `g` 入疆后开始新疆境内路线。
- `e_gk = 1` 表示第 `k` 年从枢纽 `g` 结束新疆境内路线。

目标函数应写为：

```text
min  sum_k sum_i sum_j c_ij y_ijk
   + sum_k sum_g sum_i c_gi a_gik
   + sum_k sum_i sum_g c_ig b_igk
```

其中第一项为景点间交通费用，第二项为每年入疆后到首个景点的新疆境内接驳费用，第三项为最后一个景点到离疆枢纽的新疆境内接驳费用。若不考虑具体入疆/离疆城市，可将起终点固定为乌鲁木齐；但从“新疆境内交通费尽量节省”看，更合理的是允许每年开放式进出，避免强制回到乌鲁木齐造成不必要返程。

约束：

```text
sum_k x_ik = 1                         每个普通游客可达景点恰好覆盖一次
每个年份选择一个入疆枢纽和一个离疆枢纽
每个年份形成一条开放路径
每条路径内部不得出现子回路
每年总天数不超过暑假可接受上限
只计新疆境内交通费用，不计华东至新疆的大交通费用
```

因此，区域连续性不是目标本身，而是费用最小化自然推导出的结果：如果跨区域折返很贵，模型会自动避免折返。

### 3.3 最终结果

第二问主结果采用本轮新建的 `P2_ROOTED_URUMQI_MINCOST` 两年费用最小化 MILP。该模型用高德真实驾车OD折算两人新疆境内交通费用，并约束两个暑假分别形成可执行路线、覆盖38个普通游客可达景点且不重复。

主模型假设：每年均从乌鲁木齐进入新疆境内旅游线路，并最终回到乌鲁木齐，符合现有“华东-乌鲁木齐往返”大交通数据口径。主模型为限时MILP可行解，未在360秒内证明全局最优，但已满足覆盖、连通、每年景点数和每年时长约束。

### 3.4 第二问答辩口径

第二问应明确说：我们最终没有采用“平均分景点”或“单纯谱聚类”作为主模型，而是把它修正为两条暑假线路覆盖问题，并将“新疆境内交通费用”直接放入目标函数。这样更贴合题面“新疆境内交通费用尽量节省”。

主模型中两年分别从乌鲁木齐起讫，是因为现有大交通数据以华东-乌鲁木齐往返为基础，且更保守、更容易落地。开放式入离疆模型可以作为敏感性分析，说明如果补充喀什、伊宁、阿勒泰等城市的外部航班价格，则还可以进一步优化进出疆口岸。

原 `S2_Spectral` 方案可保留为“区域聚类启发式参考”，但不应作为第二问最终主答案。

## 4. 第三问：三组文化考察路线

### 4.1 问题收束

第三问是多团队文化考察调度问题。三组人员同时开展文化考察，希望每组承担的任务量相对均衡，并且重点文化遗址得到覆盖。

本问和第一、第二问不同：楼兰古城、尼雅遗址这类普通游客不易到达或访问条件特殊的点，可以作为文化专项纳入。

### 4.2 主模型

第三问采用最小最大完成时间的 Multi-TSP/VRP 模型：

- 三个小组视为三辆车或三名旅行商。
- 每个文化景点必须分配给一个小组。
- 每个小组形成一条内部访问路径。
- 目标是最小化三组中耗时最长的一组。

定义：

- `x_ig = 1` 表示景点 `i` 分配给第 `g` 组。
- `y_ijg = 1` 表示第 `g` 组从 `i` 到 `j`。
- `T_g` 表示第 `g` 组完成时间。
- `T_max` 表示最大完成时间。

目标函数：

```text
min T_max
```

约束：

```text
T_g <= T_max
sum_g x_ig = 1                       每个文化点恰好由一组负责
每组路径连通
各组文化价值或任务量不极端失衡
```

### 4.3 最终结果

| 小组 | 景点数 | 总小时 | 天数 | 文化价值 | 费用目标 | 路线 |
|---|---:|---:|---:|---:|---:|---|
| Group1 | 2 | 98.70 | 13 | 20 | 17281.97 | 楼兰古城 -> 尼雅遗址 |
| Group2 | 7 | 90.17 | 12 | 54 | 7087.72 | 北庭故城遗址 -> 苏公塔 -> 交河故城 -> 库车王府 -> 克孜尔石窟 -> 香妃园 -> 喀什古城 |
| Group3 | 5 | 86.54 | 11 | 37 | 9095.87 | 吐峪沟麻扎村 -> 高昌故城 -> 罗布人村寨 -> 艾提尕尔清真寺 -> 石头城遗址 |

关键指标：

| 指标 | 数值 |
|---|---:|
| 最大完成时间 | 98.70小时 |
| 三组完成时间差 | 12.16小时 |
| 文化价值合计 | 111 |
| 覆盖文化专项景点 | 14个 |

### 4.4 第三问答辩口径

Group1虽然只有2个点，但楼兰古城和尼雅遗址访问难度高、交通距离长、专项考察时间长，因此总耗时最高。模型没有简单按景点数量均分，而是按“交通时间 + 考察时间 + 文化任务量”均衡，这比平均分点更符合文化考察实际。

## 5. 第四问：五一12天旅游线路与容量分配

### 5.1 问题收束

第四问是群体接待问题，不是单个游客路径问题。核心决策是：在五一12天窗口内，设计多条可售卖线路，并根据线路容量、吸引力和景区承载约束分配游客。

因此第四问采用“候选线路生成 + 容量流分配”的列生成思想：

1. 先生成若干条符合12天内游览条件的线路产品。
2. 再把游客需求分配到不同线路。
3. 每条线路不能超过沿线景区、交通和住宿的承载上限。

### 5.2 主模型

定义：

- `z_c` 表示分配到候选线路 `c` 的游客人数。
- `U_c` 表示线路 `c` 的最大接待容量。
- `a_c` 表示线路吸引力得分。
- `D` 表示五一期间总需求场景。

目标函数可以写为：

```text
max sum_c a_c z_c
```

约束：

```text
0 <= z_c <= U_c
sum_c z_c <= D
线路天数 <= 12
涉及同一景区的线路总客流不超过景区容量
```

若需求大于总容量，则模型选择吸引力较高且容量允许的线路优先承接游客。

### 5.3 最终线路分配

### 5.4 第四问答辩口径

第四问应强调“线路产品组合”而不是“唯一最优路线”。五一场景下游客偏好不同、出行时间不同，单一路线不可能服务所有人。模型通过9条线路分担客流，并让短线、伊犁草原线、喀纳斯线承担较大比例，符合节假日分流逻辑。

需要说明：景区容量与预约余量目前是仿真校准数据。如果拿到真实景区日承载、预约余票、酒店房态，本模型可以直接替换容量参数重新求解。

## 6. 四问之间的统一建模逻辑

四个问题不是四个互不相关的模型，而是同一张旅游网络图在不同决策层级上的应用：

| 问题 | 决策对象 | 图论/运筹模型 | 主要目标 | 约束重点 |
|---|---|---|---|---|
| 第一问 | 单个游客路线 | PCOP / Orienteering / 混合元启发式 | 30天内高价值低成本 | 天数、时间窗、住宿、成本 |
| 第二问 | 两年旅行计划 | 路径覆盖 / 图聚类 | 两年覆盖普通景点 | 不重复、分年、区域连续 |
| 第三问 | 三组文化队伍 | Min-Max Multi-TSP/VRP | 均衡三组任务 | 最大完成时间、文化覆盖 |
| 第四问 | 五一游客流量 | 列生成 + 容量流 | 多线路容量分配 | 景区容量、线路容量、需求 |

这条主线可以作为论文或PPT中的模型框架图：

```text
数据清洗
  -> 构建景点-交通加权图
  -> 第一问：个人路线选择
  -> 第二问：跨年份路径覆盖
  -> 第三问：多团队均衡考察
  -> 第四问：节假日容量分流
  -> 鲁棒性仿真与算法强化
```

## 7. 可放入论文的方法亮点

### 7.1 真实OD替代估算距离

最初模型如果只用经纬度距离，无法体现新疆道路绕行、山地路况和长距离接驳。现在使用高德驾车OD矩阵后，交通时间更贴近实际，尤其适合新疆这种“直线近但道路远”的区域。

### 7.2 从静态路线升级到逐日排程

仅给出景点顺序不足以说明路线可执行。本案例进一步加入：

- 每日开始时间。
- 景区开放时间。
- 单日活动时长。
- 长距离转场日。
- 住宿枢纽可用性。

这使得第一问从“路径最短”升级为“逐日可执行行程”。

### 7.3 用混合元启发式处理大规模非线性约束

严格顺序-时间联合MILP在40景点、30天尺度下会快速变得困难。最终采用 ACO + ALNS + SA 是合理选择：

- ACO适合在大图上构造初始路线。
- ALNS适合破坏和修复复杂路径。
- SA允许接受短期变差以跳出局部最优。
- 最后用逐日排程器做硬可行性验算。

因此元启发式不是“放弃严谨”，而是在保持约束校验的前提下提升全局搜索能力。

### 7.4 随机仿真用于现实扰动解释

Monte Carlo仿真引入天气、拥堵、预约失败、酒店满房等扰动。它不改变确定性最优结果，但可以告诉评委：真实旅游调度不是一次求解就结束，必须考虑扰动下的备用方案。

## 8. 当前模型仍需如实说明的限制

| 限制 | 影响 | 答辩解释 |
|---|---|---|
| 酒店房量和价格为校准数据 | 影响住宿可行性和费用 | 可接入携程/美团/酒店集团数据替换 |
| 景区容量和预约余量为仿真数据 | 影响第四问容量分配 | 可接入文旅通报或景区预约平台 |
| 12306和航班不是实时票价余票 | 影响跨城市公共交通可行性 | 当前只作为公开时刻种子 |
| Monte Carlo参数为情景设定 | 影响鲁棒性概率 | 用于敏感性分析，不声称真实概率 |
| 第一问仍有2个超标准活动日 | 影响舒适性 | 可通过减少1-2个低价值景点或增加缓冲日修复 |

这些限制不会否定模型，而是说明模型具备可扩展性。论文中应写为“参数可替换型优化框架”，而不是“实时旅游平台”。

论文正文建议这样收束：

1. 第一章或问题分析：说明四问分别对应个人路径、多年覆盖、多团队均衡、节假日分流。

5. 模型三：Min-Max Multi-TSP，给出三组文化路线与均衡指标。

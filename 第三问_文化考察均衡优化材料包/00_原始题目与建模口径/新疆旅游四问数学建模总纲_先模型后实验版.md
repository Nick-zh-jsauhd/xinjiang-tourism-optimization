# 新疆旅游线路安排四问数学建模总纲（先模型后实验版）

## 0. 这版总纲的目的

前期工作中，实验推进快于模型定型，容易造成一个严重问题：结果很多，但主模型边界、变量、目标函数和数据口径没有被优先锁定。数学建模不能用实验结果反推问题定义。

本版总纲重新按规范顺序组织：

1. 先解释题目真实决策对象；
2. 再定义统一数据图；
3. 再分别建立四问数学模型；
4. 最后才说明现有实验结果应放在哪一层。

因此，后续论文和答辩应以本文件作为主建模逻辑，现有各类实验只作为“求解器实现、灵敏度分析、鲁棒性验证和策略扩展”。

## 1. 统一数据与符号

### 1.1 节点集合

设：

- `V`：景点集合，当前清洗后共有 40 个景点；
- `V_o`：普通游客可达景点集合，排除楼兰古城、尼雅遗址等高审批/高风险点后用于第一、二问；
- `V_c`：文化考察景点集合，用于第三问；
- `H`：交通/住宿枢纽集合，例如乌鲁木齐、吐鲁番、库尔勒、伊宁、喀什、和田、阿勒泰等；
- `A`：机场节点集合；
- `R`：铁路站点集合；
- `N = V ∪ H ∪ A ∪ R`：多模式交通网络节点集合。

### 1.2 边集合与交通方式

设多模式交通图为：

```text
G = (N, E)
```

每条边 `e ∈ E` 具有：

- `mode_e`：交通方式，包含 `self_drive`、`rail`、`air`、`scenic_shuttle`、`transfer` 等；
- `τ_e`：边时间；
- `c_e`：两人交通费用；
- `ρ_e`：风险系数；
- `a_e`：可用性，表示该边在对应日期/情景下是否可用。

当前数据中的核心输入：

- `enhanced_data/multimodal_edges.csv`：多模式边；
- `enhanced_data/multimodal_nodes.csv`：多模式节点；
- `enhanced_data/enhanced_od_matrix.csv`：景点到景点的多模式最短路闭包；
- `enhanced_data/amap_driving_od_matrix_clean.csv`：真实高德驾车 OD；
- `enhanced_data/rail_12306_model_options.csv`：12306 核心铁路代表班次；
- `enhanced_data/air_public_schedule_seed.csv`：航班时刻和价格种子。

重要边界：

`enhanced_od_matrix.csv` 不是纯自驾 OD，而是多模式最短路闭包。但它仍不是完整实时班次网络。铁路和航班只是代表性样例，不等同于正式出行日期的实时余票和实时票价。

### 1.3 景点属性

对每个景点 `i ∈ V`：

- `s_i`：游览时长；
- `p_i`：门票及必要当地费用；
- `w_i`：综合游览价值；
- `[o_i, q_i]`：开放时间窗；
- `cap_i`：容量参数；
- `h(i)`：所属住宿/交通枢纽；
- `b_i`：特殊准入要求，例如边防证、审批、向导、越野车。

### 1.4 情景集合

设 `Ω` 为扰动情景集合，包括：

- 常规暑期；
- 热浪高温；
- 雨洪道路延误；
- 预约收紧；
- 暑期客流高峰；
- 复合极端冲击。

在情景 `ω ∈ Ω` 下，边时间、边费用、边风险和景点容量可变：

```text
τ_e(ω), c_e(ω), ρ_e(ω), cap_i(ω)
```

情景不是主模型的替代，而是鲁棒性验证层。

## 2. 第一问模型：30天个人旅游路线优化

### 2.1 问题本质

第一问不是单纯最短路，也不是必须游完所有景点。它本质上是：

> 在 30 天内，从可达景点中选择一组高价值景点并安排顺序、日期和交通方式，使旅游价值高、成本可控、强度可接受、时间窗可行。

因此第一问应建模为带收益的定向游问题：

```text
Prize-Collecting Orienteering Problem with Scheduling
```

如果加入逐日时间窗与住宿落点，则是：

```text
Time-Windowed Multimodal Orienteering with Lodging Feasibility
```

### 2.2 决策变量

- `x_i ∈ {0,1}`：是否游览景点 `i`；
- `y_ij ∈ {0,1}`：路线中是否从景点 `i` 转到景点 `j`；
- `z_id ∈ {0,1}`：景点 `i` 是否安排在第 `d` 天；
- `T_id ≥ 0`：第 `d` 天到达景点 `i` 的时间；
- `L_hd ∈ {0,1}`：第 `d` 晚是否住宿在枢纽 `h`；
- `u_d ≥ 0`：第 `d` 天活动强度超标松弛变量。

其中 `d = 1,...,30`。

如果直接使用 `enhanced_od_matrix.csv`，则 `c_ij, τ_ij, ρ_ij` 已表示景点 `i` 到景点 `j` 的多模式最短路闭包。若使用更严格版本，应在原始多模式图 `G=(N,E)` 上直接选择交通边，而不是只用闭包。

### 2.3 目标函数

第一问目标不应只写成费用最小，而应是价值-成本-疲劳-风险综合效用最大化：

```text
max  Σ_i w_i x_i
     - α Σ_i p_i x_i
     - β Σ_ij c_ij y_ij
     - γ Σ_ij τ_ij y_ij
     - η Σ_ij ρ_ij y_ij
     - M Σ_d u_d
```

解释：

- 第一项鼓励高价值景点；
- 门票、交通费用进入成本惩罚；
- 交通时间和风险进入疲劳/不确定性惩罚；
- `u_d` 惩罚超强度天数；
- 不同游客画像可通过改变 `β, γ, η, M` 和每日强度上限实现。

### 2.4 关键约束

1. 每个被选景点最多访问一次：

```text
Σ_d z_id = x_i
```

2. 路线连续性：

```text
Σ_j y_ij = x_i
Σ_j y_ji = x_i
```

根据是否要求从乌鲁木齐起讫，可加入 depot 起终点约束。

3. 30 天硬约束：

```text
d ≤ 30
```

4. 每日活动强度约束：

```text
Σ_i s_i z_id + 当日交通时间 ≤ B_persona + u_d
```

其中 `B_persona` 是游客画像对应的日活动阈值。

5. 景区开放时间窗：

```text
o_i ≤ T_id ≤ q_i - s_i
```

6. 住宿可行性：

每日结束位置必须能落到可住宿枢纽，或者通过额外转场到达可住宿点：

```text
Σ_h L_hd = 1
L_hd = 0, if hub h has no lodging availability
```

7. 特殊准入约束：

普通游客不可达景点不进入 `V_o`；若纳入，则需额外审批天数、向导、边防证和安全成本。

### 2.5 现有实验在第一问中的位置

现有 `HYBRID_30D_ACO_ALNS_SA` 应解释为：

> 在上述第一问模型下，使用 enhanced OD、多日排程器和硬 30 天约束进行的混合元启发式求解结果。

它不是模型本身。

当前第一问主结果：

- 32 个景点；
- 30 天；
- 总代理成本约 14548.53 元；
- 确定性排程可行；
- 仍有 2 个超强度活动日；
- 数字孪生扰动下可行率较低，说明需要缓冲/删点/拆段。

重要修正：

第一问读取的是 `enhanced_od_matrix.csv`，数据层不是纯自驾；但最终路线段中实际未选中铁路/航班，结果呈现为道路和景区接驳主导。因此论文应写成：

> 第一问使用多模式 OD 闭包作为输入，但求得的硬 30 天主路线以道路接驳为主。后续强化方向是加入长距离驾车疲劳惩罚和班次级铁路/航班边，促使模型更真实地选择跨区域公共交通。

## 3. 第二问模型：两年暑假综合交通费用最小化

### 3.1 问题本质

第二问题面强调：

> 今、明两年暑假完成新疆旅游，使新疆境内交通费用尽量节省。

因此第二问的主目标是交通费用最小化，不是景点价值最大化，也不是简单分区。

它应建模为：

```text
Two-Route Multimodal Path Covering Problem
```

其中两条路线分别代表今、明两年暑假。

### 3.2 决策变量

- `x_ik ∈ {0,1}`：景点 `i` 是否分配给第 `k` 年；
- `y_ijk ∈ {0,1}`：第 `k` 年路线是否从景点 `i` 到景点 `j`；
- `g_ak^in ∈ {0,1}`：第 `k` 年是否从口岸/枢纽 `a` 进入新疆境内路线；
- `g_bk^out ∈ {0,1}`：第 `k` 年是否从口岸/枢纽 `b` 离开新疆境内路线；
- `U_ik`：子回路消除或流模型辅助变量。

其中 `k ∈ {1,2}`。

### 3.3 目标函数

核心目标应为新疆境内综合交通费用：

```text
min Σ_k Σ_i Σ_j C_ij y_ijk
    + Σ_k Σ_a C_a,i^in g_ak^in
    + Σ_k Σ_b C_j,b^out g_bk^out
```

其中 `C_ij` 应优先来自 `enhanced_od_matrix.csv` 的多模式费用，而不是纯驾车费用。

如果使用开放式多口岸，还必须处理外部进出疆票价差异。可设外部票价溢价为 `π_k(a,b)`，得到扩展目标：

```text
min 境内交通费用 + 外部票价溢价
```

如果外部票价无法实时取得，则多口岸只能作为阈值策略：

```text
当外部票价溢价 < 乌鲁木齐起讫方案费用 - 开放式境内费用时，
开放式多口岸更优。
```

### 3.4 关键约束

1. 每个普通可达景点必须且只被安排一次：

```text
Σ_k x_ik = 1
```

2. 每年路线连续：

```text
Σ_j y_ijk = x_ik
Σ_j y_jik = x_ik
```

3. 子回路消除：

可用 MTZ、DFJ cut 或单商品流约束。

4. 暑假可执行性：

```text
Σ_i s_i x_ik + Σ_i Σ_j τ_ij y_ijk ≤ B_k
```

`B_k` 可设为暑假可接受出行上限，例如 20 至 30 天，或作为软约束进入惩罚。

5. 口岸约束：

保守版固定乌鲁木齐起讫；开放版允许喀什、伊宁、阿勒泰、和田、库尔勒等多口岸。

### 3.5 现有实验在第二问中的位置

当前第二问已有“乌鲁木齐起讫”和“开放式下界”结果，但前期主结果更多使用高德道路费用构造，因此应降级为：

> 地面交通费用基线 + 多口岸阈值敏感性实验。

第二问最终主模型应修正为：

> 基于 `enhanced_od_matrix.csv` 的多模式交通费用最小化两路径覆盖模型。

也就是说，第二问需要重跑或至少补充一版多模式费用结果，才适合称为最终主答案。

## 4. 第三问模型：三组文化考察 MinMax 均衡

### 4.1 问题本质

第三问不是普通游客游览，而是文化考察任务。核心不是费用最低，而是：

> 三组专家/学生在限定时间内完成文化考察，使各组工作量、交通时间和文化主题覆盖尽量均衡。

适合建模为：

```text
Min-Max Multi-Team Cultural Survey Routing
```

### 4.2 决策变量

- `x_ig ∈ {0,1}`：景点 `i` 是否分配给第 `g` 组；
- `y_ijg ∈ {0,1}`：第 `g` 组是否从 `i` 到 `j`；
- `T_g`：第 `g` 组总任务时长；
- `Z`：最大组任务时长。

### 4.3 目标函数

```text
min Z
```

其中：

```text
T_g = Σ_i survey_i x_ig + Σ_i Σ_j τ_ij y_ijg
T_g ≤ Z
```

可加入二级目标：

```text
min Z + λ * Var(T_g) - μ * topic_coverage
```

### 4.4 约束

- 每个文化考察点分配给一个组；
- 每组形成连通路线；
- 每组主题覆盖达到最低要求；
- 楼兰、尼雅等特殊点加入审批和安全缓冲；
- 考察时长大于普通游览时长，可设为 `survey_i = κ_i * s_i`。

### 4.5 现有实验定位

现有第三问 MinMax 结果可以作为主模型结果，因为其目标与题意较一致：不是最短游览，而是均衡文化考察负担。数字孪生缓冲实验属于鲁棒性验证。

## 5. 第四问模型：五一客流容量分配与动态分流

### 5.1 问题本质

第四问不是给个人路线，而是节假日接待组织问题：

> 面向五一客流，设计候选旅游线路并分配游客，使总接待量尽可能高，同时不超过景区、交通、住宿容量，并控制拥挤和拒载。

适合建模为：

```text
Route-Column Capacity Allocation Model
```

或：

```text
Capacitated Flow Assignment with Reservation Control
```

### 5.2 决策变量

- `r ∈ R`：候选旅游线路；
- `q_r ≥ 0`：分配到线路 `r` 的游客人数；
- `v_i`：景点 `i` 的接待游客数；
- `reject`：未能接待或需要外溢的游客量。

### 5.3 目标函数

```text
max Σ_r q_r - λ reject - μ congestion_penalty
```

也可写成成本最小：

```text
min reject + Σ_i φ_i(max(0, v_i - cap_i^safe))
```

### 5.4 约束

1. 需求守恒：

```text
Σ_r q_r + reject = demand
```

2. 景区容量：

```text
Σ_r a_ir q_r ≤ cap_i
```

3. 线路容量：

```text
q_r ≤ route_cap_r
```

4. 预约安全上限：

```text
v_i ≤ β cap_i
```

其中 `β` 可取 0.85 或 0.95，用于安全预约。

5. 动态分流：

当某景点超载或预约失败时，游客可重分配到相似主题/近邻线路：

```text
q_r' = q_r + Δ_r
```

### 5.5 现有实验定位

现有第四问容量流和动态分流实验属于模型主结果与压力测试的结合。它的数据边界必须明确：容量和预约余量为校准仿真参数，不是实时景区后台数据。

## 6. 实验结果应如何归位

现有实验不应直接作为“模型”。应按如下层级归位：

| 层级 | 作用 | 现有结果 |
|---|---|---|
| 数据层 | 构造 `G=(N,E)` 和景点属性 | `model_data/`, `enhanced_data/` |
| 主模型层 | 四问变量、目标、约束 | 本文件定义 |
| 求解层 | MILP、HiGHS、ALNS、SA、ACO、局部搜索 | 各脚本和求解输出 |
| 排程修复层 | 把路径变成逐日行程 | `hard30_route_segments.csv` 等 |
| 仿真层 | 测试扰动下是否稳定 | `digital_twin_outputs/` |
| 策略层 | 生成条件方案和应急方案 | `risk_policy_outputs/` |

换句话说：

```text
模型先行 -> 求解实验 -> 排程验证 -> 数字孪生压力测试 -> 风险策略
```

而不是：

```text
先跑实验 -> 再解释成模型
```

## 7. 论文主线应如何改写

建议论文建模章节按以下顺序重写：

1. 题意分析：四问决策对象不同，不能用一个 TSP 解决；
2. 数据与图结构：构造景点-枢纽-多模式交通图；
3. 第一问：30天带收益多模式定向游模型；
4. 第二问：两年多模式交通费用最小路径覆盖模型；
5. 第三问：文化考察 MinMax 多团队均衡模型；
6. 第四问：五一容量流与预约分流模型；
7. 算法设计：分别说明为何用 MILP/ALNS/SA/局部搜索/仿真；
8. 结果分析：实验结果只是模型求解和验证；
9. 鲁棒性与局限：说明真实数据与仿真边界。

## 8. 当前需要立刻修正的地方

### 8.1 第一问

不再说“纯高德自驾模型”。正确说法：

> 第一问读取 enhanced 多模式 OD 闭包，但最终硬30天路线以道路与景区接驳为主，未实际选中铁路/航班段。

应补充：

- 长距离驾车疲劳惩罚；
- 夜间火车/航班替代策略；
- 班次级公共交通约束。

### 8.2 第二问

不再把当前道路费用结果作为最终主模型。正确说法：

> 当前第二问结果是地面交通基线和多口岸阈值分析。最终主模型应基于 enhanced 多模式 OD 重建两年交通费用最小化。

这是最需要补做的一问。

### 8.3 第三问

模型逻辑基本成立，但应强调考察时长和审批安全缓冲，而不是普通游览。

### 8.4 第四问

模型逻辑成立，但容量、预约余量、酒店房量必须标注为仿真/校准参数，不能写成真实实时数据。

## 9. 最终总口径

本项目最终应表述为：

> 我们将新疆旅游线路安排抽象为一个多模式交通-景点-容量耦合网络。四个问题分别对应四类不同决策：第一问是30天内高价值个人路线选择，第二问是两年暑假全覆盖交通费用最小化，第三问是文化考察多团队均衡，第四问是五一容量接待与分流。每一问先建立独立数学模型，再选择相应求解算法，并通过逐日排程、数字孪生和风险策略检验现实可行性。

这才是符合数学建模规范的主线。

# 第三问：三组文化考察均衡优化材料包

本材料包整理“新疆旅游线路安排”项目第三问的输入数据、历史实验、Q3-V2 主模型结果，以及本次新增的 Q3-V3 精确求解器验证。

第三问的真实决策对象不是普通游客路线，而是“少数民族研究所三组文化考察队如何并行完成新疆文化点考察”。因此模型主线应表述为：

> 三团队文化考察 MinMax 调度：在特殊准入、现场考察时长、安全缓冲和交通时间约束下，最小化三组最大完成时间，并兼顾组间任务均衡。

## 当前结论

当前建议汇报口径：

- Q3-V2 是业务主模型：鲁棒三团队文化考察调度，包含文化候选点、特殊准入、三组 MinMax 分配、日级排程、资源日历和缓冲策略仿真。
- Q3-V3 是精确求解器验证：在 Q3-V2 的候选点和交通标签基础上，建立 Held-Karp 子集路径 + HiGHS-MILP 集合划分模型，验证当前分组在固定政策空间内精确最优。
- 14 个文化候选点中，楼兰古城与尼雅遗址固定由专项审批组执行。
- 在“特殊准入组固定、三组均乌鲁木齐起讫、资源不跨组共享”的政策空间内，最大完成时间为 `99.187` 小时，组间差距为 `9.994` 小时，最优性 gap 为 `0`。
- 该最优性不代表开放所有资源政策后的全局最优；如果允许楼兰/尼雅资源拆组共享、错日共享或改变起讫城市，需要重建模型。

## Q3-V3 精确求解器验证

位置：

```text
11_Q3_V3_精确求解器验证/
```

求解逻辑：

1. 沿用 Q3-V2 的 14 个文化候选点和多模式 OD 标签。
2. 对每个文化点子集用 Held-Karp 动态规划精确求乌鲁木齐起讫的组内最短时间闭环。
3. 将“组-子集”作为 0-1 列变量，建立三组集合划分 MILP。
4. 使用 HiGHS-MILP 做字典序精确优化：最大完成时间、组间差距、总风险、交通费。

关键输出：

- `outputs/q3_v3_exact_solver_assignment_routes.csv`
- `outputs/q3_v3_exact_solver_segments.csv`
- `outputs/q3_v3_exact_solver_column_summary.csv`
- `outputs/q3_v3_exact_solver_phase_audit.csv`
- `outputs/q3_v3_exact_solver_check.csv`
- `outputs/solve_summary.json`
- `reports/新疆旅游第三问Q3_V3精确求解器验证报告.md`

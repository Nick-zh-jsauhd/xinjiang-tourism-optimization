# 第三问模型状态与升级说明

## 当前状态

第三问已经具备可汇报主线：以文化价值和特殊准入约束筛选考察点，用三组 MinMax 均衡模型生成并行考察任务，再通过项目延期 Monte Carlo、数字孪生情景和风险画像策略给出 4-5 天缓冲建议。

当前主推表述应是：

> 三组文化专项考察 MinMax 启发式均衡解 + 5 天项目缓冲。

其中 `S3_MinMax_Group1` 专门承担楼兰古城与尼雅遗址，因为这两个点虽然数量少，但审批、远程、向导、越野和现场考察难度高。该安排符合“任务难度均衡”，不符合“景点数量均衡”，这是第三问的关键解释点。

## 不应过度表述

当前版本不建议写成以下说法：

- “精确求得全局最优三组路线”；
- “文化价值权重已经专家校准”；
- “楼兰古城、尼雅遗址可按普通旅游方式直接执行”；
- “5 天缓冲在所有极端情景下都足够”。

更准确的说法：

- 当前为启发式 MinMax 高质量可行解；
- 文化价值为规则评分，可继续专家校准；
- 特殊准入点作为文化专项考察组，不进入普通游客基准路线；
- 5 天缓冲适合均衡稳健情景，极端安全情景需机动小组或进一步延长项目周期。

## 下一轮 Q3-V2 建模建议

建议将下一轮第三问强化成以下模型：

```text
Robust Multi-Team Cultural Fieldwork Scheduling
```

核心变量：

- `x_ig`：文化点 `i` 是否分配给第 `g` 组；
- `y_ijg`：第 `g` 组是否从 `i` 转移到 `j`；
- `s_ig`：第 `g` 组访问 `i` 的开始时间；
- `r_agd`：第 `g` 组在第 `d` 天是否占用资源 `a`；
- `B`：项目缓冲天数；
- `z_omega`：情景 `omega` 下是否按时完成。

目标可以分两层：

```text
min max_g T_g + lambda * imbalance + eta * CVaR(delay_loss)
```

约束包括：

- 每个文化考察点恰好分配给一个小组；
- 审批点必须满足提前期、向导、越野车和最小团队规模；
- 每组每日工作时长不超过上限；
- 稀缺资源不能被多个小组同时占用；
- 主题覆盖要求：丝路遗址、宗教民族文化、考古遗址至少覆盖一定数量或价值；
- 鲁棒按时完成：`P(max_g T_g(omega) <= D + B) >= alpha`。

求解方法建议：

- 14 个文化点规模下，优先做 CP-SAT 或 MILP 精确校验；
- 若加入小时级时间窗与资源日历，再用 ALNS/模拟退火生成初解，CP-SAT 做局部精修；
- 保留当前启发式结果作为 warm start 和可解释基线。

# 新疆旅游铁路与航班数据采集说明

## 1. 本轮已采集的数据

本轮围绕新疆旅游线路建模所需的核心交通枢纽，补充了铁路和航班两类数据。

铁路数据使用中国铁路 12306 官方公开查询接口，查询日期为 `2026-06-20`。已覆盖 8 个核心铁路 OD：

- 乌鲁木齐 -> 吐鲁番北
- 乌鲁木齐 -> 库尔勒
- 乌鲁木齐 -> 伊宁
- 乌鲁木齐 -> 喀什
- 喀什 -> 和田
- 乌鲁木齐 -> 阿克苏
- 阿克苏 -> 库车
- 库车 -> 库尔勒

输出结果：

- `enhanced_data/rail_12306_raw_core_od.csv`：132 条原始车次记录
- `enhanced_data/rail_12306_model_options.csv`：24 条模型候选方案，每个 OD 保留最快、最便宜、夜间/早晚代表方案

航班数据使用公开页面快照整理，覆盖 8 个核心航空 OD：

- 乌鲁木齐 -> 喀什
- 乌鲁木齐 -> 伊宁
- 乌鲁木齐 -> 库尔勒
- 乌鲁木齐 -> 阿克苏
- 乌鲁木齐 -> 和田
- 乌鲁木齐 -> 库车
- 乌鲁木齐 -> 阿勒泰
- 乌鲁木齐 -> 喀纳斯

输出结果：

- `enhanced_data/air_public_schedule_seed.csv`：27 条航班时刻/报价快照记录
- `outputs/新疆旅游铁路航班数据包.xlsx`：统一工作簿，含铁路原始表、铁路模型候选表、航班种子表、来源说明

## 2. 数据可信度

铁路数据可信度较高。车次、时刻、历时、可售状态和部分票价直接来自 12306 查询接口。

航班数据可信度分层处理：

- `official_schedule_price_missing`：南航公开页面给出的航班时刻，可信度较高，但缺实时票价。
- `schedule_high_fare_proxy`：公开时刻较明确，同时有 Skyscanner/Trip.com/Ctrip 等 OTA 报价代理。
- `schedule_medium_price_missing`：公开检索片段或航班时刻页给出航班号和时间，但票价缺失。
- `fare_proxy_schedule_missing`：有 OTA 价格快照，但具体航班号/时刻仍需复核。

航班票价不是固定票价，应在论文中写为“OTA 报价快照”或“价格代理变量”，不能写成官方票价。

## 3. 建模接入方式

铁路边建议写成：

```text
rail_time = 车程 + 市内换乘时间 + 候车缓冲时间
rail_cost_for_two = ticket_price_yuan * 2
```

航班边建议写成：

```text
air_time = 飞行时间 + 机场提前到达时间 + 到达出机场时间 + 机场到市区/景区接驳时间
air_cost_for_two = fare_proxy_yuan * 2
```

本轮在 `air_public_schedule_seed.csv` 中先统一加入：

```text
airport_buffer_hours = 2.5
model_air_time_hours = duration_hours + airport_buffer_hours
```

后续如果拿到机场到景区的高德 OD，可把机场接驳时间单独建边，而不是放进固定缓冲。

## 4. 后续复核重点

1. 重新选择正式出行日期后，应重新运行 12306 抓取脚本，因为车票和余票会随日期变化。
2. 航班票价应在携程、去哪儿、航司官网或航旅纵横上按同一出行日重新截取。
3. 航班时刻应优先复核南航、乌鲁木齐航空、天津航空、成都航空等航司官网。
4. 乌鲁木齐 -> 喀纳斯、乌鲁木齐 -> 库车等季节性/支线航班要特别标注季节适用范围。
